Why Pseudocode?

High Performance

The C-based VM uses NaN-boxing, computed gotos, and register-cached stack pointers. Runs 2.5x faster than Python on recursive benchmarks.

Readable Syntax

Natural language-like constructs with clear keywords: fn, if/then/end, for/in, match/case. No cryptic symbols or complex syntax.

Rich Built-ins

80+ built-in functions including file I/O, HTTP requests, JSON parsing, dictionaries, vector operations, and cryptographic hashing.

Fast Compilation

Single-pass Pratt parser compiles source to bytecode in microseconds. No separate compilation step needed.

First-Class Functions

Define and pass functions freely. Full support for recursion, closures, and higher-order programming patterns.

Pattern Matching

Expressive match/case expressions for clean control flow. Works with numbers, strings, and complex patterns.

Code Examples

fibonacci.pseudo
// Recursive Fibonacci
fn fib(n)
    if n <= 1 then
        return n
    end
    return fib(n - 1) + fib(n - 2)
end

print(fib(30))  // 832040 in ~55ms
http_example.pseudo
// HTTP GET request
let response = http_get("https://api.example.com/data")
let data = json_parse(response)

// Work with the response
print(dict_get(data, "name"))
vectors.pseudo
// Vector operations
let data = [1, 2, 3, 4, 5]

print(vec_sum(data))     // 15
print(vec_mean(data))    // 3.0
print(vec_sort(data))    // [1,2,3,4,5]

let doubled = vec_mul(data, [2,2,2,2,2])
print(doubled)  // [2,4,6,8,10]
file_io.pseudo
// File operations
let content = read_file("data.txt")
print("Read " + str(len(content)) + " bytes")

// Write processed data
let output = upper(content)
write_file("output.txt", output)

// List directory
let files = list_dir(".")
print(files)

Technical Optimizations

Optimization Impact Description
NaN-boxing ~30% faster All values fit in 64 bits, eliminating pointer indirection
Computed gotos ~20% faster Direct threading instead of switch dispatch
Register-cached SP ~15% faster Stack pointer kept in CPU register
Superinstructions ~10% faster Fused opcodes for common patterns (GET_LOCAL_0, CONST_1, etc.)
PGO ~25% faster Profile-guided optimization with real workload data

Getting Started

1

Clone the Repository

Get the source code from GitHub.

git clone https://github.com/NagusameCS/Pseudocode.git
2

Build the VM

Compile with optimizations. Requires GCC or Clang.

cd Pseudocode/cvm && make pgo
3

Run Your First Program

Execute any .pseudo file.

./pseudo ../examples/hello.pseudo
4

Install the VS Code Extension

Get syntax highlighting and run commands.

code --install-extension pseudocode-lang-1.0.0.vsix
