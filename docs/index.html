What is Pseudocode?

Pseudocode is a high-performance programming language designed to bridge the gap between readable, intuitive syntax and native-speed execution. It compiles to bytecode executed by a highly optimized C virtual machine, with an x86-64 JIT compiler for compute-intensive sections.

Design Philosophy

  • Syntax that reads like pseudocode from a textbook
  • No semicolons, braces, or cryptic symbols
  • Keywords match natural language: if/then/end, for/in
  • Performance as close to C as possible
  • Batteries included: 80+ built-in functions

Ideal Use Cases

  • Algorithm prototyping - Test ideas quickly with clean syntax
  • Teaching programming - Syntax matches textbook pseudocode
  • Scripting & automation - Fast startup, no compilation step
  • Numerical computing - SIMD-accelerated tensor operations
  • Competitive programming - When Python is too slow

Not Designed For

  • Large-scale production systems (use Go, Rust)
  • Web development (use JavaScript, TypeScript)
  • Mobile apps (use Swift, Kotlin)
  • Systems programming (use C, Rust, Zig)

Why Pseudocode?

C-Level Performance

Direct codegen JIT achieves 1.0x C speed for numeric loops. Loop variables stay in registersâ€”no memory access in hot paths. Faster than C with volatile.

Readable Syntax

Natural language-like constructs with clear keywords: fn, if/then/end, for/in, match/case. No cryptic symbols or complex syntax.

Rich Built-ins

80+ built-in functions including file I/O, HTTP requests, JSON parsing, dictionaries, vector operations, and cryptographic hashing.

Fast Compilation

Single-pass Pratt parser compiles source to bytecode in microseconds. No separate compilation step needed.

First-Class Functions

Define and pass functions freely. Full support for recursion, closures, and higher-order programming patterns.

Pattern Matching

Expressive match/case expressions for clean control flow. Works with numbers, strings, and complex patterns.

Data Science / ML

NumPy-like tensors and matrices with SIMD acceleration (AVX2). Neural network primitives: ReLU, sigmoid, softmax, loss functions.

Cryptographic Hashing

Built-in SHA-256 and MD5 hashing, Base64 encoding/decoding. No external libraries required.

Why We're Faster Than C (volatile)

Our JIT generates tighter code than C with memory operations.

Pseudocode JIT (4 instructions/iter)

add  rbx, r14    ; x += 5
inc  r12         ; i++
cmp  r12, r13    ; i < end?
jl   loop        ; repeat

C volatile (6+ instructions/iter)

mov  eax, [x]    ; load x
add  eax, 5      ; x += 5
mov  [x], eax    ; store x
add  [i], 1      ; i++
cmp  [i], end    ; compare
jl   loop        ; repeat

Technical Achievements

Deep engineering decisions that enable C-level performance in a high-level scripting language. These techniques are rarely combined in practice due to implementation complexity.
Technique Impact Complexity Why It Matters
Direct Codegen JIT 20x+ Very High Native x86-64 code generation using DynASM. Hot loops emit 4 machine instructions per iteration with values living entirely in CPU registersâ€”no memory traffic. This is what makes us match C.
NaN-boxing ~30% High All values (numbers, booleans, pointers) encoded in a single 64-bit word using IEEE 754 NaN space. Eliminates heap allocations for primitives, removes pointer indirection, keeps everything in CPU cache.
Register-Cached VM State ~15% High Stack pointer and base pointer stored in CPU registers using GCC's register variables. Avoids memory loads/stores on every stack operationâ€”critical for tight loops.
Computed Gotos ~20% Medium Direct threaded dispatch using GCC's computed goto extension. Each opcode jumps directly to the next handler, reducing branch mispredictions from ~30% to near zero.
SIMD Tensor Ops 4x High Matrix and tensor operations use AVX2 intrinsics to process 4 doubles per instruction. Hand-tuned assembly for common patterns like matrix multiplication.
Superinstructions ~10% Medium Common opcode sequences fused into single instructions. GET_LOCAL_0, ADD_INT, FOR_COUNT, and 15+ others reduce bytecode size and dispatch overhead for hot paths.
Profile-Guided Optimization ~25% Low The VM binary is compiled twice: first to collect profiling data, then again with that data to optimize branch prediction and function inlining for real workloads.

Getting Started

Get up and running in under 2 minutes. Requires GCC/Clang and a Linux/macOS system.

1

Clone & Build

Get the source and build with PGO optimizations for best performance.

git clone https://github.com/NagusameCS/Pseudocode.git cd Pseudocode/cvm && make pgo
2

Run Hello World

Test the installation with a simple example.

./pseudo ../examples/hello.pseudo

Expected output: Hello, World!

3

Try the JIT

Run a compute-intensive benchmark to see C-level performance in action.

./pseudo -j ../benchmarks/bench_jit.pseudo

The -j flag enables direct codegen JIT for hot loops.

4

Write Your Own

Create a .pseudo file and start coding.

// myprogram.pseudo x = 0 for i in 1 to 1000000 x = x + i end print(x)

Run with: ./pseudo -j myprogram.pseudo
