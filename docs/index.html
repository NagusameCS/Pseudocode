What is Pseudocode?

Pseudocode is a high-performance programming language designed to bridge the gap between readable, intuitive syntax and native-speed execution. It compiles to bytecode executed by a highly optimized C virtual machine, with an x86-64 JIT compiler for compute-intensive sections.

Design Philosophy

  • Syntax that reads like pseudocode from a textbook
  • No semicolons, braces, or cryptic symbols
  • Keywords match natural language: if/then/end, for/in
  • Performance as close to C as possible
  • Batteries included: 80+ built-in functions

Ideal Use Cases

  • Algorithm prototyping - Test ideas quickly with clean syntax
  • Teaching programming - Syntax matches textbook pseudocode
  • Scripting & automation - Fast startup, no compilation step
  • Numerical computing - SIMD-accelerated tensor operations
  • Competitive programming - When Python is too slow

Not Designed For

  • Large-scale production systems (use Go, Rust)
  • Web development (use JavaScript, TypeScript)
  • Mobile apps (use Swift, Kotlin)
  • Systems programming (use C, Rust, Zig)

Why Pseudocode?

C-Speed Performance

x86-64 JIT compiler achieves 0.97x C speed for tight loops. The NaN-boxing VM with computed gotos runs 3x faster than Python.

Readable Syntax

Natural language-like constructs with clear keywords: fn, if/then/end, for/in, match/case. No cryptic symbols or complex syntax.

Rich Built-ins

80+ built-in functions including file I/O, HTTP requests, JSON parsing, dictionaries, vector operations, and cryptographic hashing.

Fast Compilation

Single-pass Pratt parser compiles source to bytecode in microseconds. No separate compilation step needed.

First-Class Functions

Define and pass functions freely. Full support for recursion, closures, and higher-order programming patterns.

Pattern Matching

Expressive match/case expressions for clean control flow. Works with numbers, strings, and complex patterns.

Data Science / ML

NumPy-like tensors and matrices with SIMD acceleration (AVX2). Neural network primitives: ReLU, sigmoid, softmax, loss functions.

Cryptographic Hashing

Built-in SHA-256 and MD5 hashing, Base64 encoding/decoding. No external libraries required.

How We Achieve This Performance

Technical optimizations implemented in the Pseudocode compiler and runtime. Many are unique to Pseudocode among scripting languages.
Optimization Speedup Unique? How It Works
NaN-boxing ~30% Rare All values (numbers, booleans, object pointers) are encoded in a single 64-bit word using IEEE 754 NaN space. This eliminates heap allocations for primitives and removes pointer indirection, keeping everything in CPU cache.
Computed Gotos ~20% Rare Instead of a switch statement for opcode dispatch, we use GCC's computed goto extension. Each opcode jumps directly to the next handler via a jump table, reducing branch mispredictions from ~30% to near zero.
Register-Cached SP/BP ~15% Unique The stack pointer and base pointer are stored in CPU registers using GCC's register variables. This avoids memory loads/stores on every stack operation, which happens thousands of times per second.
Superinstructions ~10% Common Common opcode sequences are fused into single opcodes. GET_LOCAL_0, ADD_INT, FOR_COUNT, and 15+ others reduce bytecode size and dispatch overhead for hot paths.
x86-64 JIT Compiler 2-3x Common Hot loops can be compiled to native machine code using DynASM. The JIT generates x86-64 instructions directly, bypassing the bytecode interpreter entirely for compute-intensive sections.
SIMD (AVX2) 4x Rare Tensor and matrix operations use AVX2 intrinsics to process 4 double-precision floats per CPU instruction. Matrix multiplication, element-wise ops, and reductions all benefit from vectorization.
Profile-Guided Optimization ~25% Common The VM binary is compiled twice: once to collect profiling data on real workloads, then again with that data to optimize branch prediction and function inlining for actual usage patterns.
String Interning ~5% Common All strings are deduplicated at parse time. Identical strings share memory, and string equality becomes a simple pointer comparison instead of character-by-character comparison.

Roadmap: Closing the Gap to C

Planned optimizations to reach our goal of <5x C performance.

Phase 1: Type Specialization

Track variable types at runtime. Generate specialized bytecode for integer-only loops, avoiding float conversions and type checks. Expected: 2-3x improvement on tight loops.

Phase 2: Inline Caching

Cache method lookups and property accesses at call sites. First call is slow, subsequent calls are near-instant. Expected: 5-10x improvement for method-heavy code.

Phase 3: Tracing JIT

Record hot execution paths and compile entire traces to native code, not just simple loops. Handle branches, function calls, and object access inline. Expected: 5-10x overall improvement.

Phase 4: Escape Analysis

Detect objects that don't escape their scope and allocate them on the stack instead of the heap. Eliminates GC pressure for temporary objects. Expected: 2x improvement for allocation-heavy code.

Getting Started

1

Clone the Repository

Get the source code from GitHub.

git clone https://github.com/NagusameCS/Pseudocode.git
2

Build the VM

Compile with optimizations. Requires GCC or Clang.

cd Pseudocode/cvm && make pgo
3

Run Your First Program

Execute any .pseudo file.

./pseudo ../examples/hello.pseudo
4

Install the VS Code Extension

Get syntax highlighting and run commands.

code --install-extension pseudocode-lang-1.0.0.vsix
